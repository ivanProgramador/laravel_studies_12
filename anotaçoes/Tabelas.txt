A migration de usuarios cria 2 tabelas 

1 - para os usuarios de fato 
2 - para as permissões desses usuarios  

então a função up da migration fica com 2 schemas

 Schema::create('users', function (Blueprint $table) {
            $table->id();
            $table->string('name',50);
            $table->string('email',100)->unique();
            $table->string('password',200);
            $table->string('role',20);
            $table->timestamps();
            $table->softDeletes();
        });

 Schema::create('users_permissions', function (Blueprint $table) {
            $table->id();
            $table->bigInteger('user_id')->unsigned();
            $table->string('permission',50);
        });

Nesse caso a segunda tabela é uma tabela auxiliar que vai fazer a ligação 
entre a entidade permissão e a entidade usuario.Embora as duas tabelas estejam
sendo criadas dentro da mesma migration, isso não significa que elas vão ter uma
relação automática por isso eu precisei estabelecer.
